## 2. فحص المدير

1. تسجيل الدخول.
2. فتح لوحة الإدارة.
3. التأكد من:
   - وجود نظرة عامة تشغيلية.
   - ظهور المستخدمين.
   - ظهور المدارس والمجموعات.
   - ظهور المحتوى المعلق للمراجعة.
4. فتح مركز المهارات والتأكد أن الإحصاءات والربط لا تزال سليمة.
5. فتح مركز الاختبارات والتأكد أن الاختبارات تظهر وأن الحفظ لا يستخدم نوافذ تنبيه قديمة.

## 3. فحص المعلم

1. تسجيل الدخول بحساب المعلم.
2. فتح لوحة العمل.
3. التأكد من:
   - ظهور فقط المسار/المادة المسندة له.
   - إمكان إضافة درس أو سؤال أو اختبار.
   - أن ما يضيفه يذهب للمراجعة ولا يظهر للعامة مباشرة.

## 4. فحص المشرف

1. تسجيل الدخول بحساب المشرف.
2. فتح لوحة العمل.
3. التأكد من:
   - ظهور المدرسة/المجموعة الخاصة به.
   - ظهور تقارير الطلاب الضعاف.
   - ظهور تقرير المدرسة.

## 5. فحص ولي الأمر

1. تسجيل الدخول بحساب ولي الأمر.
2. فتح التقارير.
3. التأكد من:
   - ظهور الطالب المرتبط به.
   - ظهور المهارات الضعيفة والتوصيات.

## 6. فحص الطالب

1. تسجيل الدخول بحساب الطالب.
2. فتح:
   - الدورات
   - الاختبارات
   - النتائج
   - التقارير
   - الخطة
3. التأكد من:
   - ظهور النتائج الفعلية.
   - ظهور الاختبارات المتاحة.
   - ظهور الدورات المتاحة.
   - أن زر "تعلم المهارة" يذهب إلى نفس صفحة التعلم الموحدة.

## 7. الفحص الآلي

يوجد الآن smoke test تشغيلي عبر:

```bash
npm --prefix server run smoke:operational:api
```

هذا الفحص يتحقق من:

- تسجيل الدخول لكل دور أساسي
- وجود التخصيص للمعلم
- ربط ولي الأمر بالطالب
- عدم ظهور المحتوى المعلق للطالب
- وجود مخزون تعلم للطالب
- وجود نتائج فعلية
- وجود تحليلات للمعلم والمشرف وولي الأمر
- وجود تقرير المدرسة

## 8. قواعد لا يجب كسرها

- المحتوى المعلق لا يظهر للطالب أو الزائر.
- صفحة "تعلم المهارة" يجب أن تذهب لمسار التعلم الموحد، لا لصفحات قديمة.
- المهارات مصدرها:
  `المسار -> المادة -> المهارة الرئيسية -> المهارة الفرعية`
- التأسيس/الموضوعات ليس بديلًا عن مركز المهارات.
- الباقات scoped حسب نوع المحتوى والمسار والمادة.
